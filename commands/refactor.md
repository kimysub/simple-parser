# 리팩토링 가이드

지정된 코드에 대해 안전한 리팩토링을 수행한다.

## 리팩토링 원칙

1. **동작 보존**: 외부에서 관찰 가능한 동작을 변경하지 않는다.
2. **단계적 진행**: 한 번에 하나의 리팩토링만 수행한다.
3. **검증 우선**: 각 단계에서 테스트가 통과하는지 확인한다.

## 실행 절차

1. 기존 테스트를 실행하여 현재 상태를 확인한다.
2. 리팩토링 계획을 수립한다:
   ```
   현재: [현재 구조 설명]
   목표: [목표 구조 설명]
   단계:
   1. [단계] → 검증: [체크]
   2. [단계] → 검증: [체크]
   ```
3. 각 단계를 수행하고 테스트가 통과하는지 확인한다.
4. 최종적으로 전체 테스트를 실행한다.

## 일반적인 리팩토링 패턴

- **함수 추출**: 긴 함수에서 의미 있는 단위를 분리
- **변수 인라인/추출**: 가독성 개선
- **조건문 단순화**: 중첩 조건을 가드 절이나 early return으로 변환
- **중복 제거**: DRY 원칙 적용 (단, 3번 이상 반복될 때만)
- **이름 변경**: 의도를 더 잘 전달하는 이름으로 변경

## 입력
$ARGUMENTS
