# 체계적 디버깅

보고된 문제를 체계적으로 조사하고 수정한다.

## 디버깅 절차

1. **재현**: 문제를 일관되게 재현할 수 있는 방법을 찾는다.
2. **격리**: 원인을 좁혀간다.
   - 관련 로그 확인
   - 입력/출력 추적
   - 이분 탐색 (git bisect 또는 코드 절반씩 주석처리)
3. **근본 원인 파악**: 증상이 아닌 원인을 찾는다.
4. **테스트 작성**: 버그를 재현하는 테스트를 작성한다.
5. **수정**: 최소한의 변경으로 수정한다.
6. **검증**: 재현 테스트가 통과하고, 기존 테스트에 회귀가 없는지 확인한다.

## 디버깅 체크리스트

- [ ] 문제를 재현할 수 있는가?
- [ ] 에러 메시지/스택 트레이스를 확인했는가?
- [ ] 최근 변경사항이 원인인가? (`git log`, `git bisect`)
- [ ] 입력 데이터가 예상 범위 내인가?
- [ ] 환경(버전, 설정)이 올바른가?
- [ ] 동일한 문제가 이전에 보고된 적 있는가?

## 입력
$ARGUMENTS
