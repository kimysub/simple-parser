# 코드 설명

지정된 코드 또는 파일의 동작을 분석하고 설명한다.

## 설명 방식

1. **고수준 개요**: 이 코드가 무엇을 하고 왜 존재하는지 설명한다.
2. **핵심 흐름**: 주요 실행 흐름을 단계별로 설명한다.
3. **의존성**: 어떤 외부 모듈/서비스에 의존하는지 설명한다.
4. **주요 설계 결정**: 왜 이런 방식으로 구현되었는지 추론한다.
5. **잠재적 문제**: 발견한 잠재적 문제나 개선 가능성을 언급한다.

## 설명 수준
- 코드를 처음 보는 개발자가 이해할 수 있도록 설명한다.
- 불필요하게 장황하지 않게, 핵심에 집중한다.
- 코드를 한 줄씩 번역하지 않고, 의도와 패턴을 설명한다.

## 입력
$ARGUMENTS
