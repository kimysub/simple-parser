# 테스트 작성

지정된 코드에 대한 테스트를 작성한다.

## 테스트 작성 원칙

1. **하나의 테스트 = 하나의 동작**: 각 테스트는 하나의 시나리오만 검증한다.
2. **AAA 패턴**: Arrange(준비) → Act(실행) → Assert(검증) 구조를 따른다.
3. **테스트 이름은 설명적으로**: 실패 시 무엇이 문제인지 이름만으로 알 수 있어야 한다.
4. **구현이 아닌 동작을 테스트**: 내부 구현이 바뀌어도 테스트가 깨지지 않아야 한다.

## 작성할 테스트 유형

### 기본 경로 (Happy Path)
- 정상 입력에 대한 정상 출력 확인

### 경계 조건 (Edge Cases)
- 빈 입력, null/undefined, 최대값, 최소값
- 컬렉션의 0개, 1개, 많은 항목

### 에러 경로 (Error Cases)
- 잘못된 입력에 대한 적절한 에러 처리 확인
- 외부 의존성 실패 시 동작 확인

### 회귀 테스트 (Regression)
- 이전에 발견된 버그가 재발하지 않음을 확인

## 절차

1. 대상 코드를 분석하여 테스트 대상을 식별한다.
2. 기존 테스트가 있으면 확인하고, 누락된 부분을 파악한다.
3. 프로젝트의 기존 테스트 패턴과 프레임워크를 따라 작성한다.
4. 테스트를 실행하여 통과하는지 확인한다.

## 입력
$ARGUMENTS
